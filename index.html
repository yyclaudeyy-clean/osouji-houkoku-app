はじめての設定手順

  1. Google Cloud Consoleで新しいプロジェクトを作成
  2. 「APIとサービス」→「ライブラリ」で Google Drive API と Google Picker API を有効化
  3. 「OAuth同意画面」で外部・Testingを選択し、アプリ名を入力。スタッフ全員のGoogleアカウントを「テストユーザー」として追加（審査不要で使えます）
  4. 「認証情報」→「認証情報を作成」→「OAuthクライアントID」（ウェブアプリケーション）を作成。このアプリを公開するURL（例：GitHub PagesのURLなど）を「承認済みのJavaScript生成元」に追加
  5. 同じ画面で「APIキー」も作成
  6. 発行されたクライアントIDとAPIキーを上の欄に貼り付けて保存

※ このファイルはブラウザ内(localStorage)にデータを保存する単体HTMLです。スタッフ全員のスマホから使うにはGitHub PagesやNetlifyなどで公開してURLを共有してください。file://（ローカルファイルを直接開く形）ではGoogleログインは使えません。
